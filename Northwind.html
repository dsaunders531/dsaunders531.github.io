Northwind

A full end-to-end web application with api, shopping basket and back-office.

The goal is to go through the process of creating a full business system with documentation. The desired outputs are tooling and patterns to make the development processes easier and faster.

Articles may appear on the wiki for the project or linked to this page.

Presently, its in the proof of concept stage and in development. It is fairly basic but new features are to be added and it will genrally improve over time.

Setting up

Prerequisites

  • Visual Studio 2017 15.7.4 (community version is OK)
  • These workloads will need adding:
    • ASP.Net and web development
    • .Net Core cross-platform development
  • These extensions will be useful (Tools->Extensions and updates)
    • Bundler and Minifier
    • CompileSass
  • ASP.Net Core SDK 2.1.302
    • https://www.microsoft.com/net/download
    • https://www.microsoft.com/net/download/thank-you/dotnet-sdk-2.1.302-windows-x64-installer
    • The version is important. There have been some changes since earlier versions and the site will not work with earlier versions.

Get the source files:

You will need to create an account on GitHub if you want to use git-bash.

The project can be downloaded in zip format. Go here https://github.com/dsaunders531/Northwind-AspNet-Core and select ‘Download Zip’ from the ‘Clone or Download’ dropdown.

GitHub command line (Git Bash)

  1. Open Git-Bash.
  2. cd to the directory where you want to download the repository.
  3. If you have not configured git-bash add your git user name and email. See the ‘configure tooling’ section in https://services.github.com/on-demand/downloads/github-git-cheat-sheet.pdf.
  4. Run git clone https://github.com/dsaunders531/Nothwind-AspNet-Core.git
  5. If you have problems with permissions, create a folder in C:\ and share it with yourself only with full control.

Install the database

The Northwind database needs to be installed before running the application.

  1. Run the script Resources/instnwnd2005_original.sql on MSSQL server or MSSQL server express.

  2. Open the project and replace all instances of [server]\\[instance] with the server and instance details for your database.

The first time you run the project any additional tables and an admin user are created.
