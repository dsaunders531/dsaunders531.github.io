Northwind 2025

Northwind 2025 uses the old Northwind database to provide data to some simple apps.

All the apps follow a microservices-type architecture. There are several simple, loosely-coupled web services and libraries.

There is no need to connect to a database!

All the data has been ported from MS Sql to an in memory dataset. In theory, you could port to any database if you wanted to (that's how its been designed). See the contents of the Data folder and the app Northwind.Migrate.CLI for reference. Entity Framework migrations and features are used to copy the data structure and data.

A simple overview of the data is provided via Northwind.Api. This is a basic JSON web api with a Swagger Docs interface for manual testing and descriptive documentation.

An api client library is available in Northwind.Api.Client.

Northwind.Web is a basic web site which provides a reference for creating paged views using React. The react build-pipeline includes Typescript. React testing is managed via Jest. Additionally, there is a business reporting framework which can handle long-running data processing tasks easily.

Authentication and Authorisation is provided by Northwind.Identity.Web this is the default ASP.Net Core identity with Identity Server. Everything here is default apart from a few OWASP reccomendations.

You may see the UI for the web app has a very vanilla appearance, these sites are for demonstrating functionality only. Since design is entirely subjective, keeping to the boostrap rules and components keeps the site experience familiar and predictable.

There are extensive unit tests for each project area, including integration tests for the api without needing an instance running.

Get the project.

Note: You need to set Northwind.Api, Northwind.Identity.Web and Northwind.Web as the startup projects. These will work best using the Kestrel (application name) server instead of IIS express.
